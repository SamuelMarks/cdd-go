# Publishing Outputs

This document details how to publish generated Client SDKs, Server Boilerplates, and OpenAPI specs emitted by `cdd-go`.

## GitHub Actions Automated SDK Publishing

You can automate publishing generated SDKs with a GitHub Action. The SDK directory generated by `cdd-go` contains its own `go.mod` (when `--no-installable-package` is not passed) and can be checked into its own repository, or pushed automatically by a script.

Here is an example `cronjob` Action that pulls an OpenAPI spec daily, generates the client, and commits/tags a new client version:

```yaml
name: Sync Client SDK
on:
  schedule:
    - cron: '0 0 * * *'
  workflow_dispatch:

jobs:
  generate:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-go@v5
      with:
        go-version: '1.25.x'

    - name: Install cdd-go
      run: go install github.com/samuel/cdd-go/cmd/cdd-go@latest

    - name: Generate SDK
      run: cdd-go from_openapi to_sdk -i ./api-spec.json -o ./sdk/

    - name: Commit & Tag Changes
      run: |
        git config user.name "github-actions[bot]"
        git config user.email "github-actions[bot]@users.noreply.github.com"
        git add ./sdk/
        git commit -m "Auto-update SDK from OpenAPI spec" || echo "No changes to commit"
        git push origin main
```
